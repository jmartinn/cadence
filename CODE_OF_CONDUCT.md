# Code of Conduct

The Cadence project is focused on building reliable, useful software. Participation is welcome from anyone willing to contribute constructively and professionally.

## Expected Conduct

Participants are expected to:

- Treat others with respect and basic courtesy.
- Keep discussions focused on the technical merits of the work.
- Offer feedback constructively and accept it with humility.
- Take responsibility for their own contributions and mistakes.
- Act in ways that support a productive environment for the project.

## Unacceptable Conduct

The following are not tolerated:

- Harassment, threats, personal attacks, or intimidation.
- Sexualized language, imagery, or advances.
- Trolling, deliberate disruption, or inflammatory remarks intended to derail productive work.
- Publishing private information about others without their explicit consent.
- Any other behavior that a reasonable person would find hostile or inappropriate in a professional software project setting.

## Enforcement

Cadence is a maintainer-led project. The maintainer has the sole discretion to decide what behavior violates this Code of Conduct and what action to take.

Reports of unacceptable behavior may be submitted:
- Privately through GitHub's vulnerability reporting (Security tab → Report a vulnerability) for sensitive matters.
- By contacting the maintainer through their public GitHub profile for other concerns.

Possible responses include warnings, editing or removal of contributions, temporary restrictions, or permanent bans from project spaces (issues, pull requests, discussions, etc.). The goal is to protect the project's focus and the ability of people to work together effectively.

## Scope

This Code of Conduct applies to all Cadence project spaces, including GitHub issues, pull requests, discussions, and any official project communications. It also applies when representing the project publicly.
